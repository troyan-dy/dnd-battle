# D&D Battler — автономная сборка на Claude Code

Готовая сборка, которая заставляет Claude Code непрерывно двигать проект вперёд:
ролевые субагенты + роадмап как единый источник правды.

> 🇷🇺 Проект адаптирован для русскоязычных разработчиков и пользователей.
> Документация, общение в `STATE.md`/`ROADMAP.md` и интерфейс продукта — на русском.
> Язык продукта по умолчанию — **русский (`ru`)**; термины D&D 2024 даются по-русски
> (см. словарь домена в [CLAUDE.md](CLAUDE.md)).

## Что внутри
```
CLAUDE.md                  # контекст проекта, который читает каждый агент (правь раздел STACK)
ROADMAP.md                 # что строим, по фазам — источник правды
STATE.md                   # журнал передачи: что произошло, что дальше, блокеры
run-loop.sh                # непрерывный цикл (headless)
.claude/
  settings.json            # опциональные хуки Stop/SubagentStop (сверь схему с докой)
  agents/
    orchestrator.md        # координатор (агент главного потока)
    pm.md                  # спецификация + критерии приёмки
    architect.md           # модель данных, протокол реалтайма, решения по безопасности
    implementer.md         # пишет код + тесты
    tester.md              # тесты, особенно корректность мультиплеер-синхронизации
    reviewer.md            # read-only контроль качества/безопасности
```

## Структура монорепо (приложение)
```
backend/                   # Python 3.12 + FastAPI (авторитетный сервер), управляется uv
  app/                     # пакет приложения (HTTP- и realtime-слои добавляются итеративно)
  pyproject.toml           # зависимости + конфигурация сборки
frontend/                  # React + Vite + TypeScript (доска на Konva — впереди)
  src/                     # исходники приложения
  package.json             # зависимости + скрипты (dev/build/lint)
```

Запуск каждого пакета:
- Backend: `cd backend && uv sync && uv run python -c "import app"` (приложение FastAPI и `/health` появятся в следующей задаче).
- Frontend: `cd frontend && npm install && npm run dev` (сборка: `npm run build`, линт: `npm run lint`).

## Установка
1. Установи Claude Code и войди. Проверь, что версия поддерживает субагентов:
   `claude --version` (см. docs.claude.com/en/docs/claude-code).
2. Скопируй эти файлы в КОРЕНЬ репозитория проекта (сохрани путь `.claude/`).
3. `git init`, если ещё не сделал — цикл опирается на git для чекпойнтов/аудита.
4. Открой `CLAUDE.md` и поправь раздел **STACK**. Backend фиксирован (FastAPI);
   фронтенд-выбор меняй, если не согласен.
5. `chmod +x run-loop.sh`

## Запуск
- Сначала интерактивно (рекомендуется): открой `claude`, затем
  `Use the orchestrator agent to run one cycle.` — посмотри, как он берёт первую задачу,
  делегирует и обновляет ROADMAP/STATE. Прогони несколько циклов вручную, чтобы выстроить доверие.
- Непрерывно: `./run-loop.sh`
  - Останавливается сам, когда в STATE.md появляется реальная запись **NEEDS HUMAN**,
    когда роадмап полностью отмечен, или после `MAX_CYCLES`.
  - Настраивается через env-переменные, например `SLEEP_SECS=60 MAX_CYCLES=50 ./run-loop.sh`.

## Почему это не съезжает с рельсов
- Одна задача за цикл, каждая — маленький проверяемый коммит → `git log` это твой аудит-трейл.
- Оркестратор не выдумывает архитектуру; рискованные области (auth, безопасность
  инвайт-ссылок, протокол синхронизации) сначала требуют агента-архитектора.
- Агенты останавливаются и спрашивают (через STATE.md → NEEDS HUMAN), а не гадают.

## Трезвая оценка
Работа часами/днями тратит реальные токены и упирается в рейт-лимиты — закладывай бюджет.
Регулярно проверяй diff; автономные циклы дрейфуют. Считай границы фаз естественными
точками для ревью перед тем, как дать ему продолжить.
